# Product Requirements Document (PRD)

## 1. 프로젝트 개요 (Project Overview)
- **프로젝트 명**: QR Code Generator
- **목표**: 사용자가 원하는 데이터(URL, 텍스트, Wi-Fi 등)를 입력하면 실시간으로 QR 코드를 생성하고 디자인을 커스터마이징하여 다운로드할 수 있는 빠르고 직관적인 웹 애플리케이션
- **기술 스택**: 
  - Framework: Next.js (App Router)
  - Styling: Tailwind CSS
  - QR Code Library: `qrcode.react` 또는 `qr-code-styling` (디자인 커스텀 고려)

## 2. 대상 사용자 (Target Audience)
- 링크나 정보를 오프라인/온라인에서 쉽게 공유하고자 하는 일반 사용자 및 소상공인
- 명함이나 포스터에 삽입할 QR 코드가 필요한 디자이너/마케터

## 3. 주요 기능 명세 (Key Features)

### 3.1. 데이터 입력 (Data Input)
- **URL/텍스트**: 기본 웹사이트 링크나 임의의 텍스트 변환 기능.
- **Wi-Fi**: 네트워크 이름(SSID), 비밀번호, 보안 유형(WEP/WPA/없음)을 입력하여 Wi-Fi 접속용 QR 생성 기능.
- **이메일**: 수신자, 제목, 본문을 미리 입력해두는 이메일 전송용 QR 생성 기능.

### 3.2. QR 코드 커스터마이징 (Customization)
- **색상 변경**: 전경색(Foreground) 구석(Eye) 색상, 배경색(Background) 지정 기능.
- **크기 및 여백 조절**: QR 코드 해상도 및 여백(Padding) 조절.
- **로고 삽입**: QR 코드 중앙에 사용자 지정 로고 이미지 삽입 기능.
- **디자인 스타일**: 점(Dot)의 모양(예: 사각형, 원형, 부드러운 모서리 등) 변경 기능.

### 3.3. 실시간 미리보기 (Real-time Preview)
- 사용자의 입력 및 커스터마이징 설정이 변경될 때마다 화면에서 즉시 QR코드 디자인이 업데이트되어 표시됨.

### 3.4. 다운로드 및 내보내기 (Export)
- 생성된 QR 코드를 이미지 파일로 다운로드.
- 지원 포맷: PNG, SVG (웹 및 인쇄용을 모두 고려).

## 4. UI/UX 디자인 요구사항 (Design Requirements)
- **반응형 디자인 (Responsive)**: 모바일, 태블릿, 데스크탑 환경에서 모두 최적화된 레이아웃 제공. (모바일 우선 접근)
- **모던하고 직관적인 UI**: 사용자가 복잡한 설명 없이도 바로 사용할 수 있는 폼과 미리보기 영역을 분리 (데스크탑: 좌/우 분할, 모바일: 상단 미리보기/하단 입력).
- **시각적 피드백**: 데이터가 비어있을 땐 기본 상태를 안내하고, 유효하지 않은 입력 시 명확한 에러 메시지 제공.

## 5. 비기능적 요구사항 (Non-functional Requirements)
- **성능**: 빠른 렌더링을 위해 가급적 모든 QR 생성 로직은 클라이언트 사이드(Client-side)에서 처리하여 서버 부하 제로화.
- **접근성(a11y)**: 스크린 리더 사용자를 고려한 적절한 폼 라벨링(`aria-labels`) 적용.
- **SEO**: 적절한 메타 태그 구성을 통한 검색 엔진 최적화.

## 6. 마일스톤 (Milestone)
- **Phase 1**: 프로젝트 초기 세팅 (Next.js + Tailwind CSS) 및 기본 URL/Text 변환 QR 코드 기능 개발.
- **Phase 2**: 다양한 데이터 타입(Wi-Fi, Email 등) 입력 폼 추가 및 실시간 미리보기 적용.
- **Phase 3**: 색상, 로고 삽입 등 디자인 커스터마이징 기능 적용.
- **Phase 4**: 이미지 다운로드(PNG/SVG) 기능 추가 및 UI/UX 폴리싱.
